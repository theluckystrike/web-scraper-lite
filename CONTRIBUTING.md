# Contributing to Web Scraper Lite

Thank you for your interest in contributing to Web Scraper Lite! This document provides guidelines and instructions for contributing.

## Code of Conduct

By participating in this project, you agree to maintain a respectful and inclusive environment for everyone.

## How to Contribute

### Reporting Bugs

If you find a bug, please report it by opening a [GitHub Issue](https://github.com/theluckystrike/web-scraper-lite/issues). Include:

- **Clear title** describing the issue
- **Steps to reproduce** the bug
- **Expected behavior** vs actual behavior
- **Screenshots** if applicable
- **Browser and extension version** info

### Suggesting Features

We welcome feature requests! Please open a [GitHub Issue](https://github.com/theluckystrike/web-scraper-lite/issues) with:

- **Clear description** of the feature
- **Use cases** - why would this be useful?
- **Mockups** or examples if applicable

### Pull Requests

1. **Fork** the repository
2. **Create a feature branch**: `git checkout -b feature/my-feature`
3. **Make your changes** and commit them: `git commit -m 'Add new feature'`
4. **Push to the branch**: `git push origin feature/my-feature`
5. **Open a Pull Request**

## Development Setup

```bash
# Clone the repository
git clone https://github.com/theluckystrike/web-scraper-lite.git
cd web-scraper-lite

# Install dependencies
npm install

# Build the extension
npm run build

# Load in Chrome
# Open chrome://extensions → Enable Developer Mode → Load unpacked
```

## Coding Standards

- Use ESLint for JavaScript linting
- Follow existing code style and conventions
- Write comments for complex logic
- Test your changes manually before submitting
